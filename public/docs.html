API Documentation

GET/health
Public

Service health check.

{ "status": "ok" }
POST/auth/login
Public

Login-or-Register: if the user exists, login; if not, auto-register then return JWT.

Request body:
{
  "phoneNumber": "13xxxxxxx",
  "password": "PlainText123",
  "name": "Optional Name"
}
Responses:
  • 201: { "token": "<jwt>", "created": true } (new user)
  • 200: { "token": "<jwt>", "created": false } (existing user)
  • 400: missing fields
  • 401: invalid credentials (existing user, wrong password)
  • 500: internal error
POST/children
Protected

Add a new child. Automatically creates parent-child relationship.

Request headers:
Authorization: Bearer <jwt_token>
Request body:
{
  "child_name": "福福",
  "gender": "男",
  "age": 8,
  "grade": "小学·二年级",
  "avatar": "https://example.com/avatar.jpg",
  "remark": "老大",
  "relation_type": 1
}
Field descriptions:
  • child_name (required): Child's name
  • gender (optional): "男" or "女" (converted to 0 or 1)
  • age (optional): Child's age
  • grade (optional): Grade string like "小学·二年级", "幼儿园大班", etc.
  • avatar (optional): Avatar URL
  • remark (optional): Parent's remark
  • relation_type (optional, default 1): 1=父亲, 2=母亲, 3=祖父母, 4=其他
Responses:
  • 201: { "message": "Child added successfully", "data": {...} }
  • 400: missing required fields
  • 401: unauthorized (no/invalid token)
  • 500: internal error
GET/children
Protected

Get all children of the current user.

Request headers:
Authorization: Bearer <jwt_token>
Responses:
  • 200: { "message": "Success", "data": [...] }
  • 401: unauthorized
  • 500: internal error
GET/children/:id
Protected

Get a specific child's details. Only accessible if the child belongs to the current user.

Request headers:
Authorization: Bearer <jwt_token>
Responses:
  • 200: { "message": "Success", "data": {...} }
  • 401: unauthorized
  • 404: child not found or access denied
  • 500: internal error
PUT/children/:id
Protected

Update a child's information. Only fields provided will be updated.

Request headers:
Authorization: Bearer <jwt_token>
Request body (all fields optional):
{
  "child_name": "福福",
  "gender": "女",
  "age": 9,
  "grade": "小学·三年级",
  "avatar": "https://example.com/new-avatar.jpg",
  "remark": "Updated remark"
}
Responses:
  • 200: { "message": "Child updated successfully", "data": {...} }
  • 400: no fields to update
  • 401: unauthorized
  • 404: child not found or access denied
  • 500: internal error
DELETE/children/:id
Protected

Remove parent-child relationship (soft delete). The child record remains in database but is no longer associated with the current user.

Request headers:
Authorization: Bearer <jwt_token>
Responses:
  • 200: { "message": "Child relationship removed successfully" }
  • 401: unauthorized
  • 404: child not found or access denied
  • 500: internal error

Notes

  • Base URL: http://localhost:3000
  • Content-Type: application/json
  • Use header: Authorization: Bearer <jwt> for protected APIs.
  • Passwords hashed with bcrypt (10 rounds).
